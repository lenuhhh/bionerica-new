# 📱 PWA Quick Reference Card\n\n## ⚡ Быстрый Старт (5 минут)\n\n### 1️⃣ Локально (Development)\n```bash\nnpm run dev\n# Откройте http://127.0.0.1:5173 в браузере\n```\n\n### 2️⃣ На Мобильном (Android)\n```\n1. Установите PWA: Меню → \"Установить приложение\"\n2. Дайте дозволи на сповіщення\n3. Тестируйте: Admin → Замовлення → Смена статуса\n```\n\n### 3️⃣ На iPhone (iOS)\n```\n1. Safari → Поділитись → На екран Додому → Додати\n2. ⚠️ iOS не дає push (Web Push API не підтримується)\n3. Push потребуватиме APNS (apple push notification service)\n```\n\n---\n\n## 🔑 VAPID Ключи\n\n### Есть ли готовые примеры?\n✅ **ДА** — в `.env` есть примеры для тестирования\n\n### Как создать свои?\n```bash\nnpm install web-push\nnpx web-push generate-vapid-keys\n# Скопируйте в .env\n```\n\n### Где их взять?\n```\n.env файл:\n  VITE_VAPID_PUBLIC_KEY=...\n  VAPID_PRIVATE_KEY=...\n```\n\n⚠️ **НЕ коммитьте .env в git!**\n\n---\n\n## 🚀 Как Тестировать Push\n\n### Локально (на экране):\n```javascript\n// F12 → Console\nnavigator.serviceWorker.ready.then(reg => {\n  reg.showNotification('Test', { body: 'Hello!' })\n})\n```\n\n### На реальном телефоне:\n1. Откройте PWA на Android\n2. Admin → Замовлення\n3. Смените статус замовлення\n4. ✅ Push должна прийти\n\n---\n\n## 📱 Что Готово ✅\n\n| Что | Статус | Где |\n|-----|--------|----|\n| Установка PWA | ✅ | Home page (banner) |\n| Offline режим | ✅ | Service Worker |\n| Local notifications | ✅ | usePush hook |\n| Admin controls | ✅ | /admin → Замовлення |\n| Newsletter | ✅ | Home + Account |\n| Referral | ✅ | Account → Loyalty |\n| **Backend push** | ❌ | Потребует Edge Functions |\n| **iOS push** | ❌ | Потребует APNS |\n\n---\n\n## 🐛 Troubleshooting\n\n| Проблема | Решение |\n|----------|----------|\n| SW не регистрируется | F12 → App → SW (должна быть зеленая галка) |\n| Push ніколи не приходять | 1) HTTPS нужен, 2) Permission должна быть granted |\n| \"VAPID key invalid\" | Проверьте .env, переинстальте web-push |\n| iOS не показует push | Нормально -
